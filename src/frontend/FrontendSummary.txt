API Integration Details:
- Schools endpoint: `POST /api/school` expects { school_name: string, school_code: string }
- Particulars endpoint: `POST /api/leave/particulars` expects { particular: string }
- Both endpoints require Bearer token authentication.
- Error responses handled and displayed in UI toast notifications.

Technical Implementation:
- Fetch API used with Authorization header for authenticated requests.
- API base URL configured via environment variable `NEXT_PUBLIC_API_BASE_URL` (defaults to http://localhost:3000).
- Loading state prevents form submission during API call.
- Modal input cleared after successful submission.
- Particulars transformation uses index as ID since API doesn't return IDs for ENUM values.
