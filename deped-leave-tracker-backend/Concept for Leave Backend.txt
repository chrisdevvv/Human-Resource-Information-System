I already have a leave module with routes, controller, and model.
This is for an employee leave tracking system.
Important business rules:
1. Employee types
- There are two employee types: teaching and non-teaching.
- Non-teaching employees gain monthly leave credits.
- Teaching employees do NOT gain monthly 1.25 leave credits every month.

2. Monthly leave credit rule
- Only non-teaching employees should receive monthly credits.
- Monthly credit for non-teaching employees:
  - +1.25 Vacation Leave (VL)
  - +1.25 Sick Leave (SL)
- Teaching employees must be excluded from monthly crediting.

3. Leave balance formula
- For every leave record, balances must be computed by the backend only.
Formula:
- bal_vl = max(0, prev_vl + earned_vl - abs_with_pay_vl - abs_without_pay_vl)
- bal_sl = max(0, prev_sl + earned_sl - abs_with_pay_sl - abs_without_pay_sl)
Meaning:
- prev_vl / prev_sl = previous balance from the latest previous leave entry of the same employee
- earned_vl / earned_sl = newly earned leave credits
- abs_with_pay_* = leave used with pay
- abs_without_pay_* = leave used without pay
- balances must never go below 0
- round balances to 2 decimal places

4. Source of truth
- Frontend must NOT be the source of truth for bal_vl and bal_sl.
- Backend must always compute bal_vl and bal_sl before saving.
- Even if frontend sends bal_vl or bal_sl, ignore them.

5. Create leave behavior
When creating a leave record:
- employee_id is required
- period_of_leave is required
- get the latest leave record of that employee
- if none exists, previous VL and SL balance start at 0
- compute bal_vl and bal_sl using the formula
- save date_of_action as server-generated current date
- save the new row

6. Update leave behavior
When updating a leave record:
- load the current leave row first
- merge old values with incoming fields
- get the previous entry of the same employee based on sequence before the current record
- recompute bal_vl and bal_sl using that previous row
- update date_of_action to current date
- save the updated row

Required behavior:
- If a leave record is updated, all later leave records of the same employee must be recomputed in sequence.
- If a leave record is deleted, all later leave records of the same employee must also be recomputed in sequence.
